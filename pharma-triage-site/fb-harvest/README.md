# fb-harvest — سجل ردود فيسبوك + رسائل العملاء (محلي، مجاني)

سكربت محلي بيبني سجل دائم لكل ردود الزملاء وكل رسائل العملاء، عشان الموقع يجاوب أي سؤال تاريخي (عدد ردود لكل زميل / لكل يوم / لكل فترة / تحليل طلب العملاء) **من غير ما يعيد مسح فيسبوك تاني**.

## ليه موجود
Graph API مابيسمحش تدوّر على المحادثات بالتاريخ — بيديك الأحدث الأول وبس. فالوصول ليوم قديم بيتطلب تقليب كل اللي بعده (بطيء + غير موثوق مع أي سقف). الحل: نحصد **مرة واحدة**، ونقرا من السجل بعدها.

## ٣ مودات — والفلو الموصى بيه للفترات الكبيرة

بما إن المشروع على خطة Spark المجانية (٢٠ألف كتابة Firestore/يوم)، الباكفيل الكبير بيضرب الكوتة لو كتبنا مباشرة. الحل: **افصل المسح عن الرفع**.

```bash
# (مرة واحدة) احفظ التوكن محليًا عشان المسح مايلمسش فايربيز خالص
node harvest.js --save-config

# 1) مسح محلي → ملف على جهازك، صفر كتابة فايربيز (مفيش كوتة). ده الجزء التقيل.
node harvest.js --local --since 2026-06-25      # من ٢٥/٦ لحد دلوقتي  (--all = من الأول)
node harvest.js --local                          # تزايدي (الجديد بس منذ آخر مسح محلي)

# 2) ارفع الملف المُنتَج لـ Firestore (resumable — لو الكوتة خلصت يكمّل بكرة)
node harvest.js --upload

# 3) انشر الموقع عشان الداشبورد يقرا الأرقام الجديدة
#    firebase deploy --only firestore:rules   (مرة واحدة للكولكشنات الجديدة)
#    firebase deploy --only hosting
```

- **`--local`** بيكتب `harvest-data.json` (بالبيانات الكاملة، فيها نص رسائل العملاء وأساميهم) — الملف ده **بيفضل على جهازك، متجاهَل من git، وماينفعش يترفع على Hosting** (لأن ملفات Hosting عامة). المسح ده صفر علاقة بفايربيز (لو عملت `--save-config` الأول).
- **`--upload`** بيرفع لـ `fbReplies` + `fbCustomerMsgs` في Firestore عشان **داشبورد الموقع (`fb-harvest.html`) يشتغل زي ما هو**. لو ضرب الكوتة اليومية بيقف بأمان ويسجّل اللي اترفع في `harvest-uploaded.json` — تعيد `--upload` بكرة فيكمّل من حيث وقف.

## مود مباشر (اختياري، للفترات الصغيرة)
```bash
node harvest.js --backfill --since 2026-07-01    # مسح → Firestore في خطوة واحدة
node harvest.js                                  # تزايدي → Firestore
```
سهل، لكن بيضرب الكوتة على الفترات الكبيرة — استخدم `--local` + `--upload` بدلًا منه.

## إعداد (مرة واحدة)
1. `npm install`
2. `serviceAccount.json` في الفولدر ده (أو في fb-classify — بيرجعله تلقائيًا). محتاجه للـ`--upload`/`--save-config`/المباشر (مش محتاجه للـ`--local` لو عملت `--save-config` قبلها).
3. `node harvest.js --save-config` — يقرا التوكن + أسماء الهاشتاجات من Firestore ويحفظهم محليًا.

## أمان وخصوصية
- **مابيلمسش `fbLog`** (سلوتات الورديات) ولا أي حاجة الموقع بيقراها للميزات الحالية.
- **Idempotent** بمفتاح رقم الرسالة → مفيش عدّ مزدوج مهما أعدت التشغيل/الرفع.
- الملف المحلي فيه بيانات عملاء (نص + أسماء) — خليه خاص. متجاهَل من git وماينزلش Hosting.

## الحقول
**fbReplies:** `{msgId, convId, ts, date, ym, agentTag, agentName, isOrder, text}`
**fbCustomerMsgs:** `{msgId, convId, ts, date, ym, custName, text, intents[], products[], keywords[]}`

## توسيع كلمات العملاء
عدّل `keywords.js` — ضيف منتجات أو كلمات نوايا وأعد `--local`. قاموس محلي مجاني، مفيش AI مدفوع.
